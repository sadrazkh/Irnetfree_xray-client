## ✨ امکانات کامل (Features)

| دسته | توضیح |
|---|---|
| **پروتکل‌ها** | VLESS (با Reality / XTLS‑Vision)، VMess، Trojan، Shadowsocks، **WireGuard** |
| **افزودن کانفیگ** | لینک تکی، چند لینک هم‌زمان، متن base64، لینک ساب‌اسکریپشن، فرم دستی WireGuard، و **چسباندن سریع با Ctrl+V** در هر جای برنامه |
| **ساب‌اسکریپشن** | افزودن، به‌روزرسانی دستی/خودکار، و نمایش **حجم مصرف و زمان باقی‌مانده با پروگرس‌بار** |
| **پروکسی محلی** | SOCKS5 + HTTP inbound با پورت‌های قابل تنظیم |
| **پروکسی سیستمی** | تنظیم خودکار پروکسی سیستم‌عامل هنگام اتصال |
| **حالت TUN** | تونل کردن **کل ترافیک سیستم** با tun2socks (Windows/macOS/Linux) |
| **روتینگ** | ساده (گلوبال/دور زدن ایران/چین/مستقیم) + قوانین سفارشی + **روتینگ پیشرفته گرافیکی** |
| **روتینگ پراسس** | هدایت ترافیک یک **برنامه‌ی خاص** به یک سرور/زنجیره دلخواه |
| **زنجیره پروکسی** | عبور ترافیک از چند سرور پشت‌سرهم (مثلاً برای دسترسی به دیتابیس از طریق WireGuard) |
| **DNS** | انتخاب از سرویس‌های آماده (Cloudflare/Google/شکن/الکترو…) یا دلخواه |
| **اشتراک LAN** | تبدیل این دستگاه به پروکسی برای بقیه دستگاه‌های شبکه |
| **کیل‌سوییچ** | قطع کامل اینترنت در صورت افت ناگهانی اتصال (جلوگیری از نشتی) |
| **به‌روزرسانی** | بررسی نسخه‌ی جدید برنامه از گیت‌هاب + نمایش نسخه‌ی هسته‌ی Xray |
| **تست** | پینگ TCP، **تأخیر واقعی** از داخل تونل، و بررسی IP/کشور خروجی |
| **آمار زنده** | سرعت و حجم دانلود/آپلود لحظه‌ای |
| **ظاهر** | تم دارک، رابط دو‌زبانه (فارسی RTL / English)، رنگ‌بندی وضعیت اتصال، نشان روی آیکون تسک‌بار، سینی سیستم (tray) |

## 🚀 نصب و اجرا (Setup)

**پیش‌نیاز:** Node.js 18+

```bash
git clone https://github.com/sadrazkh/Irnetfree_xray-client.git
cd Irnetfree_xray-client      # یا پوشه‌ی xray-client
npm install

# دانلود خودکار آخرین هسته xray-core در پوشه‌ی bin/
npm run get-xray
# یا: از داخل برنامه → تنظیمات → فایل‌های موردنیاز → دانلود

npm start
```

> نیازی به نصب دستی هسته نیست؛ اگر چیزی کم باشد، **هنگام باز شدن برنامه** پنجره‌ای ظاهر می‌شود و با اجازه‌ی شما فایل‌های لازم را دانلود می‌کند.

## ⚡ شروع سریع (Quick Start)

۱. برنامه را باز کنید؛ اگر هسته‌ی Xray نصب نیست، روی **«دانلود کن»** بزنید.
۲. به تب **سرورها** بروید و کانفیگ خود را اضافه کنید (یا هرجای برنامه **Ctrl+V** بزنید).
۳. به صفحه‌ی **اتصال** برگردید، از منوی بازشو سرور را انتخاب کنید.
۴. دکمه‌ی بزرگ روشن/خاموش وسط را بزنید.
۵. وقتی **تولبار و لوگو سبز** شد یعنی متصل‌اید ✅ — برای اطمینان «بررسی IP» را بزنید.

## 🔗 افزودن کانفیگ‌ها (Adding Configs)

در تب **سرورها** روی «+ افزودن» بزنید و یکی از موارد زیر را بچسبانید (برنامه نوعش را خودکار تشخیص می‌دهد):

**۱) لینک تکی:**
```
vless://uuid@example.com:443?security=reality&pbk=...&sni=...#MyServer
```

**۲) چند لینک با هم (هر خط یک کانفیگ):**
```
vless://...#Server-1
vmess://...
trojan://...@host:443#Server-3
ss://...#Shadowsocks-1
```

**۳) متن base64 (خروجی خام ساب):** کل بلوک base64 را بچسبانید، خودکار دیکد و وارد می‌شود.

**۴) لینک ساب‌اسکریپشن:** هر خطی که با `http(s)://` شروع شود به‌عنوان ساب اضافه و دریافت می‌شود (به بخش بعد).

**۵) WireGuard دستی:** دکمه‌ی «+ WireGuard» → فرم را پر کنید.
> 💡 فیلد **Endpoint** باید آدرس عمومی سرور به شکل `host:port` باشد (مثل `de.example.com:51820`)، نه آدرس محلی تونل. فیلد **Address** آدرس محلی شما و حتماً `/32` است (مثل `10.8.0.2/32`). برای دسترسی به یک رنج خاص، آن را در **Allowed IPs** بگذارید (مثل `192.168.60.0/24`) یا `0.0.0.0/0` را نگه دارید.

> ⚡ **میان‌بر:** هرجای برنامه (بیرون از کادرهای متنی) **Ctrl+V** بزنید تا کانفیگ یا لینک ساب از کلیپ‌بورد فوراً اضافه شود.

ویرایش: روی آیکون ✎ هر سرور بزنید تا پروتکل، ترنسپورت، TLS/Reality، کلیدها و … را تغییر دهید.

## 📡 ساب‌اسکریپشن (Subscriptions)

در تب **ساب‌ها**:

- **افزودن ساب:** آدرس `https://...` و یک نام دلخواه بدهید.
- **به‌روزرسانی:** دستی (آیکون ⟳) یا **خودکار** با فاصله‌ی دلخواه (مثلاً هر ۶۰ دقیقه).
- **حجم و زمان:** اگر سرور ساب هدر استاندارد `Subscription-Userinfo` بفرستد، روی کارت ساب دو نوار نمایش داده می‌شود:
  - 🔵 **حجم مصرف:** «۱۲٫۳ GB / ۵۰ GB · ۲۵٪» با نوار رنگی (سبز → زرد → قرمز با نزدیک شدن به سقف).
  - 🟢 **زمان باقی‌مانده:** «۱۸ روز مانده» یا «منقضی شده».

> با حذف یک ساب، سرورهای متعلق به آن هم پاک می‌شوند؛ سرورهایی که دستی اضافه کرده‌اید دست‌نخورده می‌مانند.

## 🔌 حالت اتصال: پروکسی یا TUN (Connection Mode)

روی کارت حالت در صفحه‌ی اتصال بزنید تا بین دو حالت سوییچ کنید:

| حالت | چه‌چیزی تونل می‌شود؟ | نیاز به ادمین | کاربرد |
|---|---|---|---|
| ⚡ **پروکسی** | فقط برنامه‌هایی که از پروکسی سیستمی/SOCKS استفاده می‌کنند (مرورگرها، …) | خیر | سبک و سریع، اکثر کارهای روزمره |
| 🛡️ **TUN** | **کل ترافیک سیستم** (همه‌ی برنامه‌ها) | بله | وقتی برنامه‌ای پروکسی را نمی‌خواند یا تونل کامل لازم است |

> در حالت TUN روی ویندوز به فایل‌های `tun2socks` و `wintun.dll` و **دسترسی ادمین** نیاز است. اگر ادمین نباشید، برنامه پیشنهاد می‌دهد خودش را با دسترسی ادمین دوباره اجرا کند.
> روی مک، هنگام اتصال یک‌بار پنجره‌ی **رمز عبور** نمایش داده می‌شود و تونل کامل برپا می‌شود.

## 🧭 روتینگ (Routing)

تب **روتینگ** چهار سطح کنترل می‌دهد:

### ۱) روتینگ ساده (Simple)
یکی از حالت‌ها را انتخاب کنید:
- **گلوبال:** همه از پروکسی (به‌جز شبکه‌ی محلی).
- **دور زدن ایران:** ترافیک داخلی مستقیم، بقیه از پروکسی.
- **دور زدن چین:** مشابه، برای سایت‌های چین.
- **مستقیم:** همه‌چیز مستقیم (برای تست).
به‌علاوه: **بلاک تبلیغات** و **Sniffing** (تشخیص دامنه برای روتینگ دقیق‌تر).
> حالت‌های «دور زدن» و «بلاک تبلیغات» به فایل‌های `geoip.dat` و `geosite.dat` نیاز دارند (از بخش فایل‌های موردنیاز دانلود کنید).

### ۲) قوانین سفارشی (Custom rules)
هر خط به شکل `نوع , مقدار , خروجی`:
```
domain, geosite:google, proxy
ip, 8.8.8.8, direct
domain, geosite:category-ads-all, block
port, 443, proxy
```
خروجی‌ها: `proxy` / `direct` / `block`.

### ۳) روتینگ پیشرفته گرافیکی (Advanced)
سوییچ «روتینگ پیشرفته» را روشن کنید و قانون بسازید. هر قانون: **نوع → مقدار → مقصد**.

- **نوع:** `IP` / `دامنه` / `پورت` / `پروسه`.
- **مقدار:**
  - برای IP و دامنه یک **لیست پیشنهادی geoip/geosite** هست؛ لازم نیست حفظ باشید — تایپ کنید یا انتخاب کنید (مثل `geoip:ir`، `geosite:google`، `geosite:openai`).
  - مثال‌های دستی: `1.2.3.0/24`، `example.com`، `443`.
- **مقصد:** یک **منوی قابل‌جستجو** (مثل صفحه‌ی هوم) که همه‌ی کانفیگ‌ها، زنجیره‌ها، `مستقیم` و `بلاک` را نشان می‌دهد — برای لیست‌های طولانی، کافی است اسم را تایپ کنید.
- **بقیه‌ی ترافیک:** مقصد پیش‌فرض برای هر چیزی که با قانون‌ها مطابقت نکند.

قانون‌ها **از بالا به پایین** بررسی می‌شوند و **اولین تطابق** برنده است. سپس روی صفحه‌ی اتصال گزینه‌ی 🧭 را انتخاب و وصل شوید.

**مثال:** ترافیک اوپن‌ای‌آی از سرور آمریکا، سایت‌های ایرانی مستقیم، بقیه از سرور آلمان:
```
دامنه   geosite:openai     →  US-Server
دامنه   geosite:category-ir →  مستقیم
(بقیه‌ی ترافیک)            →  DE-Server
```

### ۴) روتینگ بر اساس پروسه (Process)
نوع قانون را روی **«پروسه»** بگذارید و از لیست برنامه‌های در حال اجرا یکی را انتخاب کنید (دکمه‌ی ⟳ برای تازه‌سازی لیست).

> ℹ️ **چطور کار می‌کند و محدودیت‌ها:** هسته‌ی Xray امکان تطبیق مستقیم «نام پروسه» را ندارد. بنابراین برنامه IPهای مقصدی را که آن پروسه به آن‌ها وصل می‌شود کشف کرده و آن‌ها را به مقصد انتخابی شما هدایت می‌کند (و در حافظه نگه می‌دارد). یعنی:
> - برای برنامه‌هایی با **سرور ثابت** (بازی، کلاینت اختصاصی، ابزار دیتابیس) بهترین نتیجه را دارد.
> - برنامه‌هایی که به IPهای مشترک CDN وصل می‌شوند (مثل مرورگر) قابل‌اعتماد نیست.
> - با روشن‌کردن «به‌روز نگه‌داشتن مسیر پروسه‌ها»، هر چند ثانیه مقصدهای جدید اضافه و هسته کوتاه ری‌لود می‌شود (پیش‌فرض خاموش = فقط هنگام اتصال). دکمه‌ی «پاک‌سازی IPهای یادگرفته‌شده» هم هست.

**مثال:** فقط ترافیک یک بازی (`game.exe`) از سرور با پینگ پایین، بقیه مستقیم.

## ⛓️ زنجیره پروکسی (Proxy Chains)

در تب **زنجیره** یک زنجیره بسازید: نام بدهید و چند سرور را **به‌ترتیب** بچینید (با کشیدن‌و‌رها کردن جابه‌جا می‌شوند). ترافیک از سرور اول → دوم → … → خروجی عبور می‌کند. هر زنجیره مثل یک «کانفیگ» در صفحه‌ی اتصال و در روتینگ پیشرفته قابل‌انتخاب است.

> 🔎 برای لیست بلند سرورها، بالای «سرورهای موجود» یک **کادر جستجو** هست و لیست **اسکرول‌دار** است.

**کاربرد رایج — دسترسی به دیتابیس از طریق WireGuard:** اگر WireGuard مستقیم وصل نمی‌شود، یک کانفیگ معمولی را **اول** و WireGuard را **آخر** زنجیره بگذارید؛ سپس در روتینگ پیشرفته رنج IP دیتابیس را به این زنجیره بفرستید:
```
IP   10.20.0.0/16   →  ⛓ DB-Chain   (Config → WireGuard)
```

## 🌐 اشتراک در شبکه محلی (LAN)

سوییچ «اجازه به شبکه محلی» را روشن کنید تا این دستگاه به‌عنوان پروکسی برای بقیه‌ی دستگاه‌های شبکه عمل کند.

- به‌محض روشن‌کردن، **آدرس و پورت** نمایش داده می‌شود، مثل:
  `آدرس برای دستگاه‌های دیگر: 192.168.1.20:10809 (HTTP) • 192.168.1.20:10808 (SOCKS)`
- روی دستگاه دیگر، همان IP و پورت را به‌عنوان پروکسی HTTP یا SOCKS وارد کنید.
- روی ویندوز برای باز شدن **فایروال** به **دسترسی ادمین** نیاز است (برنامه خودش پورت‌ها را باز می‌کند).
- پس از تغییر این گزینه، یک‌بار دوباره وصل شوید تا اعمال شود.

## 🚨 کیل‌سوییچ (Kill Switch)

سوییچ «کیل‌سوییچ» را روشن کنید تا اگر اتصال **به‌طور ناگهانی** قطع شد، **کل اینترنت بسته شود** و هیچ ترافیکی بدون تونل نشت نکند.

- هنگام قطع ناگهانی، یک **بنر قرمز** ظاهر می‌شود با دو دکمه:
  - **اتصال مجدد** — دوباره وصل می‌شود.
  - **باز کردن اینترنت** — مسدودسازی را برمی‌دارد و به حالت عادی برمی‌گردد.
- قطع **عمدی** توسط خودتان کیل‌سوییچ را فعال نمی‌کند.
- روی ویندوز به **دسترسی ادمین** نیاز دارد (با فایروال کار می‌کند). اگر برنامه بسته یا کرش شود، قانون فایروال خودکار پاک می‌شود تا اینترنت دائمی بسته نماند.

## 🔐 اجرای با دسترسی ادمین (Run as Admin)

حالت TUN، کیل‌سوییچ و باز کردن فایروال LAN روی ویندوز نیازمند دسترسی ادمین‌اند. سه راه:
1. ترمینال/برنامه را **Run as administrator** اجرا کنید، سپس `npm start`.
2. از داخل برنامه دکمه/پیام **«اجرا با دسترسی ادمین»** را بزنید (خودش بسته و دوباره با ادمین باز می‌شود).
3. (برای تست توسعه، بدون ساخت exe) از PowerShell:
   ```powershell
   Start-Process ".\node_modules\electron\dist\electron.exe" -ArgumentList "<مسیر-کامل-پروژه>" -Verb RunAs
   ```

## 📦 فایل‌های موردنیاز (Required Files)

در **تنظیمات → فایل‌های موردنیاز**، هر مؤلفه با یک کلیک دانلود/به‌روزرسانی می‌شود (بدون ساخت دوباره‌ی برنامه):

| مؤلفه | کاربرد |
|---|---|
| **هسته Xray** | اجرای اصلی پروکسی/تونل (نسخه‌اش هم نمایش داده می‌شود) |
| **tun2socks** | حالت TUN |
| **wintun.dll** | حالت TUN (فقط ویندوز) |
| **geoip + geosite** | روتینگ مبتنی بر کشور/دسته (دور زدن ایران/چین، بلاک تبلیغات، قوانین `geoip:`/`geosite:`) |

- برنامه به‌طور خودکار معماری درست (x64/arm64) را می‌گیرد و روی مک باینری‌ها را برای اجرا روی Apple Silicon امضای ad‑hoc می‌کند.
- دکمه‌ی **«حذف فایل‌های دانلودشده»** همه‌ی فایل‌هایی که برنامه دانلود کرده (در پوشه‌ی داده‌ها) را پاک می‌کند (نسخه‌ی همراهِ نصب دست‌نخورده می‌ماند).

## 🆙 نسخه و به‌روزرسانی (Version & Update)

در **تنظیمات → درباره و به‌روزرسانی**:
- **نسخه‌ی برنامه** نمایش داده می‌شود.
- دکمه‌ی **«بررسی به‌روزرسانی»** نسخه‌ی فعلی را با آخرین ریلیز گیت‌هاب مقایسه می‌کند؛ اگر نسخه‌ی جدیدی باشد، دکمه‌ی **«دانلود»** صفحه‌ی ریلیز را باز می‌کند.
- **نسخه‌ی هسته‌ی Xray** هم در همین صفحه و کنار «فایل‌های موردنیاز» نشان داده می‌شود.

## 📊 آمار و تست (Stats & Testing)

- **پینگ TCP:** زمان handshake تا سرور (سریع).
- **تأخیر واقعی:** یک نمونه‌ی موقتی Xray بالا می‌آید و تأخیر واقعی از داخل تونل اندازه گرفته می‌شود (برای زنجیره هم کار می‌کند).
- **بررسی IP:** IP و کشور خروجی (مستقیم یا از طریق پروکسی).
- **آمار زنده:** سرعت و حجم دانلود/آپلود لحظه‌ای + مجموع این نشست.
- **تست همه:** پینگ هم‌زمان همه‌ی سرورها/زنجیره‌ها.

## 🎨 ظاهر و رابط کاربری (UI)

- **رنگ‌بندی وضعیت:** هنگام اتصال، **تولبار و لوگو سبز** (با ✓) و هنگام قطع، **آبی با ✕ قرمز** می‌شوند.
- **نشان روی تسک‌بار ویندوز:** آیکون برنامه در نوار وظیفه نشان سبز (متصل) یا قرمز (قطع) می‌گیرد.
- **دکمه‌های پنجره:** کوچک‌کردن، **بزرگ‌نمایی/بازگردانی** (دکمه‌ی □)، و بستن به سینی سیستم (✕).
- **سینی سیستم (Tray):** نمایش/خروج/قطع اتصال از منوی tray.
- **زبان:** فارسی (RTL) و English، با یک کلیک.

## 🏗️ ساخت نسخه نصبی (Build)

**ویندوز:**
```bash
npm run dist          # NSIS installer + portable در پوشه‌ی dist/
```

**مک (Intel + Apple Silicon) — فقط روی خودِ macOS:**
```bash
npm run gen-icons     # یک‌بار، اگر assets/icon.icns نباشد
npm run dist:mac          # هر دو معماری
npm run dist:mac:x64      # فقط Intel
npm run dist:mac:arm64    # فقط M1/M2/M3/M4
```
خروجی: `IRNetFree-<version>-x64.dmg` و `IRNetFree-<version>-arm64.dmg` (و `.zip`).
> ⚠️ بیلد مک روی ویندوز ساخته نمی‌شود (به `hdiutil`/`codesign` نیاز است).

**اولین اجرا روی مک:** چون بدون گواهی Apple امضا شده، یک‌بار روی برنامه **راست‌کلیک → Open** بزنید یا `xattr -cr /Applications/IRNetFree.app`.

**لینوکس:**
```bash
npm run dist:linux    # AppImage + deb
```

## 🔒 امنیت (Security)

- `contextIsolation` فعال و `nodeIntegration` غیرفعال؛ رندرر فقط از طریق preload با main حرف می‌زند.
- هنگام خروج/قطع، پروکسی سیستمی، روت‌های TUN، قانون LAN و کیل‌سوییچ **خودکار پاک‌سازی** می‌شوند.
- کلیدها و کانفیگ‌ها فقط **محلی** ذخیره می‌شوند؛ چیزی به سروری ارسال نمی‌شود (به‌جز خود ساب‌ها که شما اضافه می‌کنید).

## 🛠️ عیب‌یابی (Troubleshooting)

| مشکل | راه‌حل |
|---|---|
| **«هسته Xray پیدا نشد»** | تنظیمات → فایل‌های موردنیاز → دانلود (یا `npm run get-xray`). |
| **وصل شد ولی اینترنت ندارید** | حالت روتینگ را روی «گلوبال» بگذارید؛ پورت‌ها و سرور را چک کنید؛ «بررسی IP» را بزنید. |
| **حالت TUN کار نمی‌کند (ویندوز)** | برنامه را با **ادمین** اجرا کنید؛ مطمئن شوید `tun2socks` و `wintun.dll` نصب‌اند. |
| **TUN روی مک کار نمی‌کند** | هنگام اتصال رمز عبور را بدهید؛ tun2socks را از «فایل‌های موردنیاز» دانلود کنید؛ پیش از خروج، اول **قطع** کنید. |
| **روتینگ geoip/geosite بی‌اثر است** | فایل‌های `geoip.dat` و `geosite.dat` را دانلود کنید. |
| **LAN کار نمی‌کند** | برنامه را با ادمین اجرا کنید (فایروال)؛ روی دستگاه دیگر IP:پورت درست را بزنید؛ یک‌بار دوباره وصل شوید. |
| **کیل‌سوییچ فعال نمی‌شود** | با **ادمین** اجرا کنید (با فایروال کار می‌کند). |
| **اینترنت بعد از کرش بسته ماند** | برنامه را دوباره باز کنید؛ هنگام شروع، قانون کیل‌سوییچ پاک می‌شود. |
| **ساب دریافت نمی‌شود** | آدرس باید با `http(s)://` شروع شود و در دسترس باشد. |
| **پروکسی سیستمی پاک نشد** | برنامه را دوباره باز کنید؛ هنگام شروع پاک‌سازی می‌شود. |
